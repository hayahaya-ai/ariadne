Date: 2026-07-10

## Method

This sweep reran the exact eight repositories and pinned commits used for the
`v0.2.0-rc.1` calibration. Git status was clean and every repository HEAD was
rechecked before scanning. The candidate binary was built from the local source
tree at the implementation commit above. No repository code, hook, dependency
installer, test, action, or workflow was executed.

For every target the candidate emitted an `ariadne.verdict/v2` JSON verdict and
a gated text verdict. The same target file also produced consolidated JSON and
JSONL fleet scans. Results were compared with the source-level classifications
recorded in the first sweep. These are Ariadne calibration results, not
vulnerability claims about the sampled repositories.

## Red-first benchmark

Seven paired fixture worlds were added before production changes, growing the
Story Lab from 50 to 57 fixtures. The required red run measured:

- verdict accuracy: 50/57 (87.7%);
- data-egress precision 13/15 (86.7%), recall 13/13;
- mutable-tool precision/recall 6/6;
- secret-path precision 14/16 (87.5%), recall 14/14.

The seven failures covered generic VS Code runtime discovery, unresolved-runtime
precedence, valid multiline YAML, instruction-to-ordinary-CI composition,
deterministic issue automation, and repository-write/private-data conflation.

After the corrections, `make eval` reports 57/57 verdict accuracy and 100%
precision and recall for all three supported exposure families.

## Correctness closure

The second sweep confirms the six product gaps from the first audit are closed
for the exact pinned corpus:

1. shared editor paths require supported runtime keys;
2. `hardened` requires a runtime and at least one enforced control;
3. valid multiline YAML parses without accepting unterminated collections or
   interpreting shell block contents as YAML structure;
4. trust inputs, managed workflows, and authorities compose only through
   runtime/source-compatible occurrences;
5. repository integrity authority is distinct from private-data authority;
6. mutable privileged workflow/action references receive no inaccurate
   data-egress claim or remedy while a dedicated supply-chain family is absent.

The public sweep produced no new false positive, false inconclusive, or unsafe
inverse regression within the audited corpus. Together with the 57-world paired
benchmark and full verification suite, this clears the Phase 5.1 correctness
bar for an RC2 pull request.
